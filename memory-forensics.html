Detect Malicious Processes in Memory Dumps

Introduction to Memory Forensics

Beginner 45 minutes

Introduction

Memory forensics is a critical aspect of digital forensics that focuses on analyzing computer memory (RAM) dumps. This analysis can reveal active processes, network connections, and malware that may not be visible through traditional disk forensics.

In this course, you will learn how to analyze memory dumps to identify malicious processes, understand process hierarchies, and extract valuable forensic artifacts from memory.

Memory Acquisition

Before analysis can begin, memory must be properly acquired. Common tools include:

  • FTK Imager - GUI-based tool for memory acquisition
  • DumpIt - Lightweight command-line tool
  • WinPmem - Open-source memory acquisition tool

Important Note

Memory must be acquired as quickly as possible after an incident to preserve volatile data. The longer you wait, the more evidence may be lost.

Acquisition Steps:

  1. Identify the target system
  2. Select the appropriate acquisition tool
  3. Connect write-protected storage media
  4. Execute the acquisition tool with proper parameters
  5. Verify the integrity of the acquired memory dump

Analysis with Volatility

Volatility is the primary framework for memory analysis. Key plugins include:

# List all processes
volatility -f memory.raw --profile=Win10x64 pslist

# View network connections
volatility -f memory.raw --profile=Win10x64 netscan

# Dump process memory
volatility -f memory.raw --profile=Win10x64 memdump -p 1234 -D output/

# List loaded DLLs for a specific process
volatility -f memory.raw --profile=Win10x64 dlllist -p 1234

# Extract command history
volatility -f memory.raw --profile=Win10x64 cmdscan

Common Volatility Plugins:

Plugin Description Use Case
pslist Lists processes Initial enumeration
psscan Scans for processes, including hidden ones Finding hidden processes
pstree Shows process tree Understanding process relationships
netscan Shows network connections Identifying C2 communications

Detecting Malware in Memory

Common indicators of malware in memory dumps:

  • Unusual process names or paths
  • Suspicious parent-child process relationships
  • Hidden processes
  • Unexpected network connections
  • Injected code in legitimate processes
  • Suspicious loaded modules or DLLs
  • Unusual memory regions with execute permissions
  • Hooks in system APIs

Warning

Malware can use various techniques to hide its presence in memory. Always cross-reference multiple indicators and use multiple detection methods.

Common Malware Techniques:

Process Hollowing

Malware creates a legitimate process in a suspended state, replaces its memory with malicious code, then resumes execution.

DLL Injection

Malware forces a legitimate process to load a malicious DLL, allowing code execution within that process context.

Direct Kernel Object Manipulation (DKOM)

Malware directly modifies kernel structures to hide processes from standard enumeration methods.
