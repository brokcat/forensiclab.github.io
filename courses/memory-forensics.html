1. Introduction à l'analyse de mémoire

Complété

L'analyse de mémoire forensique est une technique essentielle dans l'investigation numérique qui permet d'examiner le contenu de la mémoire vive (RAM) d'un système. Contrairement aux données stockées sur le disque dur, la mémoire contient des informations volatiles qui disparaissent lorsque l'ordinateur est éteint.

Pourquoi l'analyse de mémoire est importante:
  • Détection de malwares avancés qui opèrent uniquement en mémoire
  • Récupération de clés de chiffrement et mots de passe
  • Analyse des processus en cours d'exécution
  • Identification des connexions réseau actives
  • Récupération de fichiers supprimés encore présents en mémoire
Point clé: La mémoire volatile contient souvent des informations cruciales qui ne sont pas disponibles sur le disque dur.

2. Acquisition de la mémoire

Complété

L'acquisition de la mémoire est la première étape cruciale de l'analyse forensique. Elle doit être effectuée avec soin pour préserver l'intégrité des données.

Outils d'acquisition de mémoire:
Outil Plateforme Description
FTK Imager Windows Outil commercial avec capacités d'acquisition de mémoire
DumpIt Windows Outil léger pour créer rapidement une image mémoire
LiME Linux Module kernel pour l'acquisition de mémoire Linux
OSXPmem macOS Outil d'acquisition pour systèmes macOS
Attention: L'acquisition de mémoire modifie nécessairement l'état du système. Minimisez l'impact en utilisant des outils légers et en documentant vos actions.

3. Analyse avec Volatility

Complété

Volatility est un framework open-source puissant pour l'analyse de mémoire forensique. Il prend en charge de nombreux systèmes d'exploitation et offre une large gamme de plugins pour extraire différents types d'informations.

Commandes Volatility essentielles:
# Identifier le profil du système d'exploitation
python vol.py -f memory.dmp imageinfo

# Lister les processus en cours d'exécution
python vol.py -f memory.dmp --profile=Win10x64_19041 pslist

# Afficher les connexions réseau actives
python vol.py -f memory.dmp --profile=Win10x64_19041 netscan

# Extraire les clés de registre
python vol.py -f memory.dmp --profile=Win10x64_19041 hivelist

# Analyser les DLLs chargées par un processus
python vol.py -f memory.dmp --profile=Win10x64_19041 dlllist -p 1234
Conseil: Créez toujours une timeline des événements en combinant les résultats de différents plugins Volatility pour obtenir une vue complète de l'activité du système.
